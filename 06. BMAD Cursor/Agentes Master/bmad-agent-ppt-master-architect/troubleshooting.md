# Presentation Troubleshooting

Systematically diagnose and fix presentation problems across design, readability, narrative, branding, data visualization, and technical issues.

## Activation

When the user reports a problem or asks for help fixing a presentation issue, follow this diagnostic process:

### Step 1 — Problem Identification

Categorize the issue into one or more domains:

| Domain | Symptoms |
|--------|----------|
| **Text Excess** | Slides look like documents, audience reads instead of listens, too many bullets |
| **Narrative Weakness** | No clear story, slides feel disconnected, unclear purpose |
| **Design Inconsistency** | Mixed fonts, colors, styles across slides, no visual system |
| **Low Readability** | Small text, poor contrast, cluttered layout, unreadable from distance |
| **Cluttered Slides** | Too many elements, no white space, visual noise |
| **Branding Issues** | Inconsistent logo use, wrong colors, template not followed |
| **Data Overload** | Raw tables pasted, too many charts, no insight headline |
| **Hierarchy Problems** | All text same size, no visual distinction between title/body/footnote |
| **Font Issues** | Mismatched fonts, decorative fonts in corporate context, size inconsistency |
| **Unprofessional Visual** | Clip art, low-res images, excessive animations, dated design |

### Step 2 — Root Cause Analysis

For each identified problem, determine:

1. **Problem** — What is visually/structurally wrong?
2. **Root Cause** — Why does this happen? (common causes below)
3. **Impact** — How does it affect the presentation's effectiveness?
4. **Fix** — Specific, actionable solution
5. **Validation** — How to confirm the fix works

### Common Problems & Solutions

**Text Excess**
- **Cause:** Content pasted from documents without synthesis
- **Fix:** Apply the "1 message, 3-5 bullets, 8-12 words each" rule. Move detail to speaker notes. Split dense slides into multiple focused slides.
- **Validate:** Can each slide be understood in under 10 seconds?

**No Narrative Flow**
- **Cause:** Slides created in isolation without storyline planning
- **Fix:** Define the narrative framework first (see storytelling-structure capability). Rearrange slides to follow logical progression. Add transition slides between sections.
- **Validate:** Read only the slide titles in sequence — do they tell a coherent story?

**Inconsistent Design**
- **Cause:** Multiple authors, no template, copy-paste from other decks
- **Fix:** Apply a Slide Master with defined layouts. Standardize fonts (2 families max), colors (brand palette), and spacing. Reformat all slides to use template layouts.
- **Validate:** Print all slides as thumbnails — do they look like they belong together?

**Poor Readability**
- **Cause:** Small fonts, low contrast, busy backgrounds
- **Fix:** Enforce minimum sizes (28pt titles, 18pt body). Test contrast ratio (4.5:1 minimum). Remove or simplify background elements. Increase white space.
- **Validate:** View slides from 3 meters / 10 feet distance — can you read everything?

**Cluttered Slides**
- **Cause:** Trying to fit too much on one slide
- **Fix:** Apply the "one idea per slide" principle. Remove decorative elements that don't add information. Increase margins and spacing. Split into multiple slides.
- **Validate:** Can you identify the focal point of each slide instantly?

**Data Overload**
- **Cause:** Excel tables pasted directly, too many data points
- **Fix:** Replace tables with charts. Use insight headlines ("Revenue grew 23%" not "Revenue Chart"). Show 3-5 key data points, move detail to appendix. Choose the right chart type for the data story.
- **Validate:** Does each data slide have a clear "so what" message?

**Branding Problems**
- **Cause:** No brand guidelines followed, template ignored, manual formatting
- **Fix:** Obtain and apply the corporate template. Set brand colors in theme colors. Use approved fonts. Position logo per brand guidelines. Enforce consistency with Slide Master.
- **Validate:** Would the brand/design team approve this deck?

**Animation & Transition Excess**
- **Cause:** Using animations for entertainment rather than communication
- **Fix:** Remove all non-essential animations. Keep only: appear for sequential reveals, fade for soft transitions between slides. Never use spinning, bouncing, or flying effects in corporate context.
- **Validate:** Does every animation serve a communication purpose?

### Step 3 — Prioritized Action Plan

Deliver fixes in priority order:

```
## Diagnosis Summary

### Critical Fixes (Do First)
1. [Problem] → [Fix] — Impact: [Why this matters most]

### High-Priority Fixes
1. [Problem] → [Fix] — Impact: [Expected improvement]

### Polish Items (If Time Permits)
1. [Problem] → [Fix]

### Estimated Effort
- Quick fixes (under 30 min): [list]
- Medium effort (1-2 hours): [list]
- Significant rework (half day+): [list]
```

### Step 4 — Offer Implementation

After diagnosis, offer to:

1. **Rewrite** problematic slides with improved content
2. **Restructure** the narrative flow
3. **Specify** exact design changes (fonts, colors, sizes, layouts)
4. **Create** a style guide for the deck
5. **Generate** a corrected version of specific slides

## Quick Diagnostic Checklist

Run this rapid assessment for any presentation:

- [ ] Every slide has a clear, informative title
- [ ] No slide has more than 5 bullet points
- [ ] No bullet exceeds 12 words
- [ ] Fonts are consistent throughout
- [ ] Colors follow a defined palette
- [ ] All charts have insight headlines
- [ ] White space is adequate
- [ ] Reading order is logical
- [ ] Branding is consistent
- [ ] Opening slide captures attention
- [ ] Closing slide drives action
