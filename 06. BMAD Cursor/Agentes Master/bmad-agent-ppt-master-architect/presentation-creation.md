# Presentation Creation

Create complete presentations from scratch — from initial concept to fully structured slide deck with content, design recommendations, and speaker notes.

## Activation

When the user wants to create a new presentation, follow this process:

### Step 1 — Gather Essential Context

If not already provided in the conversation, collect:

1. **Objective** — What is the presentation's purpose? (inform, persuade, propose, report, train, pitch)
2. **Audience** — Who will see this? (board, executives, team, clients, partners, external)
3. **Type** — What kind of presentation? (executive, status report, commercial proposal, institutional, technical, pitch, workshop, training, board deck, roadmap, results, strategy)
4. **Content Source** — Is there existing content (documents, data, notes) or building from scratch?
5. **Constraints** — Slide count limit? Time limit? Corporate template? Deadline?
6. **Style** — Preferred style? (executive/minimalist, data-heavy, visual/image-rich, technical/detailed)

If the user provides enough context upfront, skip redundant questions and proceed.

### Step 2 — Define Narrative Structure

Based on the context, propose a storyline using the appropriate narrative blocks:

**Executive/Board:**
Context → Key Message → Supporting Data → Implications → Decision Required → Next Steps

**Status Report:**
Period Overview → Achievements → Metrics/KPIs → Challenges → Mitigation → Outlook

**Commercial Proposal:**
Context/Pain Point → Solution → Differentiators → Case Studies → Investment → ROI → Next Steps

**Technical:**
Problem Statement → Current State → Proposed Solution → Architecture/Design → Implementation → Timeline → Risks

**Pitch:**
Hook → Problem → Solution → Market → Traction → Team → Ask

**Training/Workshop:**
Objectives → Agenda → Core Content (modular) → Exercises → Summary → Resources

Present the proposed structure and refine with user feedback before proceeding.

### Step 3 — Generate Slide Content

For each slide, use the standard generation format:

```
Slide X — Title
Message: [Core message]
Bullets:
- [Key point 1]
- [Key point 2]
- [Key point 3]
Visual Suggestion: [Layout, chart, image, icon recommendation]
Objective: [What this slide achieves]
Speaker Notes: [Talking points]
```

**Content rules:**
- Maximum 3-5 bullets per slide
- Each bullet: one clear idea, 8-12 words maximum
- Titles: 4-8 words, action-oriented when possible
- One core message per slide
- Data slides: one chart/visual per slide with a clear takeaway headline

### Step 4 — Design Recommendations

After content generation, provide:

1. **Color palette** — Based on branding or neutral corporate default
2. **Font pairing** — Title font + body font recommendation
3. **Layout patterns** — Which layouts to use for different content types
4. **Visual assets** — Icon, image, and chart suggestions
5. **Consistency rules** — Spacing, margins, alignment standards

### Step 5 — Delivery Options

Offer the user:

- **Executive summary version** (fewer slides, higher synthesis)
- **Detailed version** (complete content, appendix material)
- **Speaker notes** for each slide
- **Handout adaptation** (more text-dense for leave-behind)

## Quality Checklist

Before finalizing, verify:

- [ ] Every slide has one clear message
- [ ] Narrative flows logically from beginning to end
- [ ] Visual consistency across all slides
- [ ] No text-heavy slides without synthesis
- [ ] Audience-appropriate language and depth
- [ ] Branding adherence (if applicable)
- [ ] Strong opening and strong closing
- [ ] Call-to-action or next steps clearly defined
