# Refactor

Melhore código existente sem alterar comportamento.

## Processo

1. **Leia o código** atual completo
2. **Identifique problemas** — Complexidade, duplicação, acoplamento, performance
3. **Apresente plano** — Antes/Depois com justificativa
4. **Implemente** — Edite os arquivos
5. **Compare** — Mostre ganhos (linhas, complexidade, performance)
6. **Valide** — Lint/typecheck + testes existentes devem continuar passando
7. **Responda no formato Executor** — `references/standard-output.md`

## Entregáveis
- Código refatorado
- Comparação antes/depois
- Ganhos (métricas quando possível)
