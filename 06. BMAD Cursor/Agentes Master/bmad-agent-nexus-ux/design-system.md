# Design System

Crie ou atualize design systems, tokens e padrões.

## Processo

1. **Leia tokens existentes** — Cores, tipografia, spacing, breakpoints
2. **Identifique gaps** — Tokens faltando, inconsistências
3. **Proponha** — Novos tokens ou ajustes com justificativa
4. **Implemente** — Crie/edite arquivos de tokens/theme
5. **Documente** — Catálogo de componentes, usage guidelines
6. **Valide** — Consistência visual, contraste A11y
7. **Responda no formato Executor** — `references/standard-output.md`

## Entregáveis
- Tokens criados/atualizados
- Documentação de uso
- Checklist de contraste
