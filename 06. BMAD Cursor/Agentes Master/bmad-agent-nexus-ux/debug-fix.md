# Debug & Fix

Diagnostique e corrija bugs visuais e de comportamento UI.

## Processo

1. **Leia o código** do componente/página com problema
2. **Diagnostique** — Identifique a causa raiz
3. **Apresente diagnóstico** com evidência
4. **Corrija** — Edite o(s) arquivo(s) diretamente
5. **Valide** — Lint/typecheck e, se possível, teste
6. **Verifique A11y** — Garantir que o fix não quebra acessibilidade
7. **Responda no formato Executor** — `references/standard-output.md`

## Entregáveis
- Diagnóstico com causa raiz
- Arquivo(s) corrigido(s)
- Validação (lint/types)
