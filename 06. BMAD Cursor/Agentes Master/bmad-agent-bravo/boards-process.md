# Boards & Process

Customize Boards, Process Templates e Inherited Processes.

## Processo

1. **Entenda a necessidade** — Novo processo, customização ou migração de processo?
2. **Carregue** `references/playbook-boards.md` para padrões
3. **Projete o TO-BE** — WIT types, states, transitions, fields, rules
4. **Prefira Inherited Process** sobre XML (on-prem) quando possível
5. **Documente** o de-para (AS-IS → TO-BE)
6. **Sugira salvar** mapeamento no sidecar (SM)

## Entregáveis
- Definição de Process Template (states, transitions, fields, rules)
- Mapeamento AS-IS → TO-BE
- Checklist de validação
