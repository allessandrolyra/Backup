# Licensing Analysis

## Purpose

Analyze, compare and recommend Microsoft 365 licenses based on user requirements.

## Process

1. **Gather context** — Ask the user:
   - How many users? SMB or Enterprise?
   - Which country/region? Which cloud/tenant?
   - CSP, EA or other channel?
   - Which workloads are needed? (Teams, Copilot, Purview, Security, etc.)
   - Regulatory requirements?
   - Current licensing (if upgrade/migration)?

2. **Analyze base license needs** — Evaluate:
   - Primary SKU fit
   - Included rights and services
   - Limitations and partially included features
   - Plan family eligibility (Business ≤300 users, Enterprise above)

3. **Evaluate add-ons** — For each additional need:
   - Eligibility against base SKU
   - Dependencies and prerequisites
   - Overlap with base license features
   - Cost-benefit of add-on vs. upgrading base SKU

4. **Cost optimization** — Consider:
   - Consolidation opportunities (combining SKUs)
   - Waste identification (features paid but unused)
   - Mix-and-match strategies (different SKUs for different user profiles)
   - Contract term impact on pricing

5. **Deliver recommendation** using the standard response format:
   - Executive Summary
   - Detailed Analysis with comparison table
   - Resource classification (included, add-on, not available)
   - Cost comparison
   - Final recommendation with justification
   - Official sources with validation date

## Key Rules

- Always validate plan family eligibility before recommending
- Differentiate channel vs. contract vs. commercial model vs. enrollment
- Never assume Commercial Cloud availability equals other clouds
- Clearly separate confirmed facts from consulting interpretation
- Cite source hierarchy: Product Terms > Licensing Docs > Service Descriptions > Learn > Partner Center > Pricing Pages
