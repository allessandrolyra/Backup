PASSO 2 - TOKENS DE ACESSO

================================================================================
PASSO 3 - VALIDACAO (DRY RUN)
================================================================================

3.1 Execute primeiro o script de dry-run para validar a configuracao:
    > devopsmigration execute -c "Script Migracao - DryRun.txt"

    Este script tem o Processor com "Enabled": false e Serilog em "Verbose".
    Ele VAI conectar nos endpoints e validar tipos/campos, mas NAO migra nada.

3.2 Verifique no log:
    - Nao deve haver linhas [ERR] (erro critico)
    - Warnings [WRN] sobre Microsoft.VSTS.Build.FoundIn e IntegrationBuild
      sao aceitaveis (campos de sistema com allowed values diferentes)
    - Confirme que a mensagem final NAO diz "Some work item types or their
      fields are not valid"

3.3 Se houver erros, corrija antes de prosseguir para o Passo 4.

================================================================================
PASSO 4 - EXECUCAO DA MIGRACAO
================================================================================

4.1 Execute o script de migracao real:
    > devopsmigration execute -c "Script Migracao.txt"

4.2 Monitore o log em tempo real para verificar:
    - Work items sendo criados com sucesso no target
    - Links e attachments migrando corretamente
    - Nenhum erro de campo ou estado

================================================================================
PASSO 5 - MIGRACAO DE TEST PLANS E TEST SUITES (OPCIONAL)
================================================================================

Test Plans e Test Suites foram excluidos da migracao principal porque nao
suportam o campo Custom.ControleMigracao (tipos de sistema). Para migra-los:

5.1 Crie um segundo script de configuracao com:
    - ProcessorType: "TfsTestPlansAndSuitesMigrationProcessor"
    - Configure TestPlanQuery ou TestPlanIds conforme necessario
    - Documentacao: https://devopsmigration.io/docs/how-to/migrating-plans-and-suites/

================================================================================
RESUMO DAS ALTERACOES NO SCRIPT
================================================================================

O que esta configurado no "Script Migracao.txt":

1. WIQL FILTRADA
   - AND [System.WorkItemType] NOT IN ('Test Plan', 'Test Suite')

2. TfsWorkItemTypeValidatorTool
   - ExcludeWorkItemTypes: Test Plan, Test Suite, Shared Steps, Shared Parameter
   - NENHUM campo excluido (migracao completa)

3. MAPEAMENTO DE ESTADOS (System.State) - FieldValueMap
   - Bug: New->Novo, Active->Em Execucao, Resolved->Done, Closed->Closed
   - Epic: Resolved->Done (demais iguais)
   - Feature: Resolved->Done (demais iguais)
   - Task: Aguardando homologacao/aprovacao->Active (demais iguais)
   - User Story: Rascunho->New, Trabalho em andamento->Active,
     Pronto para testes->Committed, Testando->Committed,
     Pronto->Done, Concluido->Closed, Cancelada->Removed

4. MAPEAMENTO DE VALORES - FieldValueMap
   - ValueArea (Feature/User Story): Architectural->Arquitetura, Business->Negocios
   - Priority (User Story): valores 5-8 mapeados para 4

5. MAPEAMENTO DE CAMPOS - FieldToFieldMap
   - Bug: Custom.Causaraiz -> Custom.CausaRaiz
   - Feature: Custom.1e5d61de-... -> Custom.AreasImpactadas
   - Feature: Custom.7d9f22e8-... -> Custom.AreaSolicitante
   - Feature: Custom.50cccf96-... -> Custom.PontuacaoWSJF
   - Feature: Custom.9e0e1ef6-... -> Custom.RequisitosNaoFuncionais
   - Feature: Custom.627bab8c-... -> Custom.Beneficios
   - Feature: Custom.f68e53a0-... -> Custom.CriteriosDeAceite
