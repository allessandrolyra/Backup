# Improve Delivery Flow

Review the broader delivery system and recommend improvements that reduce toil, increase traceability, and raise deployment confidence.

## Focus Areas

- environment standardization
- work item traceability
- approval flow and governance
- reuse through templates and shared patterns
- DORA-aligned flow improvements
- observability feedback after deploy

## Process

1. Understand the current delivery operating model
2. Identify bottlenecks, manual steps, and repeated failure patterns
3. Recommend standards, templates, and workflow improvements
4. Separate quick wins from structural improvements
5. Highlight changes that require team alignment

## Output

Return:
- quick wins
- medium-term improvements
- governance and standardization actions
- success signals to measure improvement

## Autonomous Use

This capability can be used in autonomous audits that consolidate recommendations across `iac/`, `pipelines/`, and `config/`.
