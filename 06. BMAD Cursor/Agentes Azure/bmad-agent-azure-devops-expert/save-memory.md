# Save Memory

Persist the current session context for future DevOps guidance.

## Process

1. Read `{project-root}/_bmad/_memory/azure-devops-expert-sidecar/index.md`
2. Update the current context with:
   - active project or repository focus
   - relevant Azure DevOps, GitHub, IaC, and environment standards
   - newly discovered team preferences or patterns
   - approved recommendations and next steps
3. Check whether `patterns.md` should record a reusable convention or recurring issue
4. Check whether `chronology.md` should record a significant event, incident, or milestone
5. Write condensed updates back to memory files

## Output

Confirm save with a short summary of what was updated and why it matters for future delivery decisions.
