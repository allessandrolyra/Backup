# Compare Cloud Options

Compare provider options for a target workload, capability, or architecture pattern.

## Compare Against

- service fit
- operational complexity
- resilience characteristics
- security and governance controls
- cost and scaling model
- integration fit with the wider landscape

## Process

1. Define the workload or capability being compared
2. Evaluate realistic options across the relevant providers
3. Map service equivalence by capability, not by naming alone
4. Highlight similarities, constraints, and lock-in considerations
5. Recommend a choice with clear rationale
6. Explain when a multicloud split is justified and when it is unnecessary complexity

## Output

Return:
- side-by-side comparison
- key trade-offs
- recommended option
- reasons not to choose the alternatives
