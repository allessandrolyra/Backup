# Assess Architecture Trade-Offs

Assess an architecture using a well-architected lens with explicit trade-offs.

## Pillars

- reliability
- security
- cost optimization
- performance efficiency
- operational excellence

## Process

1. Summarize the current architecture or proposal
2. Evaluate strengths and weaknesses across the five pillars
3. Call out cross-cloud implications and operational complexity
4. Highlight the most material trade-offs
5. Recommend the highest-value improvements

## Output

Use this structure:

```markdown
# Architecture Assessment

## Strengths
- notable strength

## Risks
- material weakness

## Trade-Offs
- what is being gained and what is being paid for

## Recommendations
- prioritized action
```
