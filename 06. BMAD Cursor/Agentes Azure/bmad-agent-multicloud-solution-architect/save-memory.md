# Save Memory

Persist the current architecture context for future sessions.

## Process

1. Read `{project-root}/_bmad/_memory/multicloud-solution-architect-sidecar/index.md`
2. Update the current context with:
   - active system scope and architectural goals
   - relevant constraints, dependencies, and cloud posture
   - approved patterns, decisions, and preferred outputs
   - important next steps for architecture work
3. Update `patterns.md` if a reusable standard or diagram convention emerged
4. Update `chronology.md` if a significant review or decision occurred
5. Write condensed updates back to memory files

## Output

Confirm save with a short summary of what was updated and why it matters for future architecture decisions.
