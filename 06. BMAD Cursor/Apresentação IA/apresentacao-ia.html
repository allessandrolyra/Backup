Bloco 1 — O que é IA?

Mas afinal, o que é IA?

🧒 IA é como uma criança que aprende por exemplos.

Você mostra milhares de fotos de gatos, e ela aprende a reconhecer gatos sozinha.
Você mostra milhões de textos, e ela aprende a escrever textos sozinha.
A máquina aprende com dados e melhora com a prática.

Ninguém programa regra por regra. A IA descobre os padrões sozinha.

⚙️
Nota técnica: Inteligência Artificial é o campo da ciência da computação que desenvolve algoritmos capazes de realizar tarefas que normalmente requerem inteligência humana: reconhecimento de padrões, tomada de decisão e geração de conteúdo.

Bloco 2 — LLMs

O que são Tokens?

O modelo não lê palavras inteiras. Ele lê pedaços chamados tokens.

O gato pul ou no muro

Cada cor é um token. Uma palavra pode ser 1 token ou vários pedaços.

💰 Modelos pagos cobram por token. Quanto mais tokens na conversa, mais custa.

⚙️
Nota técnica: Tokens são gerados por tokenizadores como BPE (Byte-Pair Encoding) ou SentencePiece. Um token equivale a ~¾ de uma palavra em inglês. O contexto do modelo (ex: 128K tokens) determina quanta informação ele "lembra" numa conversa.

Bloco 2 — LLMs

Prompt: A Arte de Perguntar Bem

Quanto melhor a pergunta, melhor a resposta.

❌ Prompt vago

"Faz um código aí"

→ Resultado genérico, sem contexto
✅ Prompt claro

"Crie uma função em Python que recebe uma lista de nomes e retorna apenas os que começam com 'A', ordenados alfabeticamente"

→ Resultado preciso e utilizável

💡 Prompt = a pergunta ou instrução que você dá para a IA.

⚙️
Nota técnica: Prompt Engineering é a técnica de otimização de instruções para maximizar a qualidade da saída do modelo. Inclui estratégias como few-shot learning (dar exemplos), chain-of-thought (pedir raciocínio passo a passo) e system prompts (definir contexto e regras).

Bloco 3 — IDEs com IA

Primeiro: O que é uma IDE?

IDE é o programa onde desenvolvedores escrevem código.

Pense nela como o Microsoft Word, mas para programação.
O mais conhecido: Visual Studio Code (VS Code).

IDE = Integrated Development Environment = Ambiente de Desenvolvimento Integrado

⚙️
Nota técnica: IDE integra editor de código, compilador/interpretador, debugger e ferramentas de build em uma interface única. Exemplos: VS Code (Electron/TypeScript), IntelliJ (JVM), Xcode (Apple). IDEs com IA adicionam LLMs como copiloto integrado ao fluxo de desenvolvimento.

Bloco 4 — Agentes de IA

O que Forma um Agente?

4 peças de um quebra-cabeça

🧠

Cérebro (LLM)

O modelo de linguagem que já vimos — é o que faz ele "pensar"

🔧

Ferramentas

Pode ler arquivos, executar comandos, editar código, acessar sistemas

💾

Memória

Lembra do que foi feito em sessões anteriores — não começa do zero

🎭

Persona

Tem nome, personalidade e regras de como agir e se comunicar

⚙️
Nota técnica: Arquitetura de agente = LLM (raciocínio) + Tool Use (execução de funções) + Memory (persistência de estado) + System Prompt (persona). O agente opera em ciclo observe-think-act, invocando ferramentas conforme necessário até concluir a tarefa.

Bloco 6 — BMAD Method

O que é uma Metodologia?

🍰 É como uma receita de bolo.

Um jeito organizado de fazer as coisas, com etapas definidas.
Se pular passos, o bolo não cresce.

BMAD = Build More, Architect Dreams
"Construa mais, arquitete sonhos"

⚙️
Nota técnica: BMAD é um framework AI-native para delivery de software que orquestra agentes especializados em fases sequenciais com gates de qualidade. Ciclo principal: Analysis → Planning → Solutioning → Implementation. Versão atual: 6.2.1.

Bloco 6 — BMAD Method

Por que é Diferente?

💾

Memória

Os agentes lembram das decisões anteriores. Não precisa repetir contexto toda vez.

📄

Rastreabilidade

Cada fase gera um documento que a próxima fase usa. Tudo conectado e documentado.

🛡️

Qualidade

Tem pontos de validação antes de avançar. Não pula etapas, não acumula problemas.

⚙️
Artifacts: Documentos estruturados (PRD, Architecture Doc, Epic/Story files) persistidos em _bmad-output/ — cada artefato alimenta a próxima fase. Quality Gates: Implementation Readiness Check (pré-sprint), Story Validation (pré-dev), Code Review (pós-dev), Adversarial Review (segurança).
