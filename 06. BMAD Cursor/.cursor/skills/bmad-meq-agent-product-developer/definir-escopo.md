# Definir Escopo

Guie o usuário na definição do escopo do produto através de perguntas estruturadas.

## Processo

1. **Entenda o produto** — O que será construído? Web, Android, iOS ou multiplataforma?

2. **Perguntas essenciais (Jobs-to-be-Done):**
   - Qual problema resolve? (job do usuário)
   - Para quem? (persona/usuário-alvo)
   - Por que agora? (urgência/contexto)
   - O que o usuário usa hoje como alternativa? (competidor ou workaround)
   - Quais são as features essenciais para o MVP?

3. **Plataformas** — Para cada plataforma (web, Android, iOS):
   - O escopo é idêntico ou há diferenças?
   - Prioridade de lançamento?

4. **Documente** — Resuma decisões. Sugira criar Product Brief (CB) ou PRD (PR) quando houver clareza suficiente.

5. **Memória** — Se decisões importantes foram tomadas, sugira salvar (SM).

## Regras

- Foque em valor, não em tecnologia
- MVP primeiro — o mínimo que valida a ideia
- Web, Android e iOS podem ter escopos diferentes; priorize com critério
