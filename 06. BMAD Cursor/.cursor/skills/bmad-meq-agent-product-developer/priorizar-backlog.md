# Priorizar Backlog

Ajude o usuário a priorizar features usando critérios de valor e impacto.

## Processo

1. **Liste as features** — O que está no backlog? (pode vir da memória ou da conversa)

2. **Critérios de priorização:**
   - **Valor** — Quanto resolve o problema do usuário?
   - **Impacto** — Quantas pessoas beneficiam?
   - **Esforço** — Complexidade relativa (baixo/médio/alto)
   - **Dependência** — O que precisa vir antes?
   - **Risco** — O que falhar seria crítico?

3. **Matriz valor/esforço** — MoSCoW opcional: Must / Should / Could / Won't — Sugira ordenação: alto valor + baixo esforço primeiro.

4. **Plataformas** — Para web + Android + iOS:
   - Mesma prioridade em todas ou lançamento por fases?
   - Ex: Web MVP → Android → iOS

5. **Documente** — Atualize a ordem no backlog. Sugira salvar (SM).

## Regras

- Priorização é decisão de produto, não técnica
- MVP = menor conjunto que entrega valor testável
- Revisar prioridades a cada ciclo
